## README conversion checklist

- Clear one-line value proposition
- Badges for license, stars, forks, last commit, Python, FastAPI, and React
- Screenshot or social preview image
- Quick-start commands near the top
- Architecture diagram
- Support matrix
- Security model
- Roadmap link
- Contribution link
- Community link
- Conservative legal disclaimer

## Launch checklist

1. Confirm no secrets or private files are committed.
2. Run the release safety gate.
3. Verify README links.
4. Add repository topics.
5. Add a social preview image.
6. Create a GitHub Release with clear release notes.
7. Pin the repository on the TradeNodeX GitHub profile.
8. Open GitHub Discussions.
9. Add labels for issue triage.
10. Publish a concise launch post.

## Public messaging rules

Use:

- self-hosted infrastructure
- audit-first visibility
- operator control
- API integrations
- security-conscious deployment
- open-source control plane

Avoid:

- performance guarantees
- return claims
- passive income language
- managed service claims
- unauthorized third-party logo usage
- screenshots containing private data
