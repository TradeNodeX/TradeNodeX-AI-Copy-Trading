# TradeNodeX Roadmap

This roadmap describes the direction of the open-source edition. It is not a commitment to specific dates. Priorities may change based on security, reliability, community feedback, and exchange API changes.

## Current focus

- Improve open-source documentation quality
- Strengthen deployment guidance
- Improve exchange connector validation notes
- Expand test coverage around routing and worker behavior
- Improve operator observability and audit clarity

## Near-term roadmap

### v0.4 Documentation and operator onboarding

- Better README structure
- Architecture documentation
- Security checklist
- Community and contribution guides
- Issue and pull request templates
- Demo screenshots and visual assets

### v0.5 Deployment hardening

- Clearer Docker and cloud deployment notes
- Production environment checklist
- Reverse proxy notes
- Backup and restore notes
- Operational runbook draft

### v0.6 Connector validation

- Exchange-by-exchange validation checklist
- Credential field matrix
- Testnet and demo environment notes
- Private stream diagnostics
- REST fallback documentation

### v0.7 Observability improvements

- Queue health notes
- Worker status documentation
- Audit-log query examples
- Error taxonomy and troubleshooting guide

### v1.0 Stable self-hosted release baseline

- Stable installation path
- Complete operator guide
- Security policy coverage
- Validated release process
- Documented limitations and compatibility matrix

## Community input wanted

The project welcomes feedback in these areas:

- Deployment environments
- Exchange compatibility gaps
- Documentation clarity
- Testing strategy
- Security hardening
- Operator workflow improvements

Please open a GitHub Discussion for open-ended ideas and an Issue for reproducible bugs or tightly scoped feature requests.
